<task type="auto">
  <name>Task 1: Configure wxt.config.ts for Safari and run Safari build</name>
  <files>wxt.config.ts</files>
  <action>
    First, verify whether `wxt-module-safari-xcode` is a viable option:

    ```bash
    pnpm info wxt-module-safari-xcode version 2>/dev/null || echo "not found"
    ```

    **If the module is found and has a version compatible with WXT 0.20.x:**
    Add to `wxt.config.ts`:
    ```typescript
    import { defineConfig } from 'wxt';

    export default defineConfig({
      manifest: { /* existing manifest config unchanged */ },
      modules: ['wxt-module-safari-xcode'],
      safariXcode: {
        projectName: 'LinkedInHHRRScreener',
        bundleIdentifier: 'com.linkedin-hhrh.screener',
        appCategory: 'public.app-category.productivity',
      },
    });
    ```
    Then: `pnpm add -D wxt-module-safari-xcode` and run `pnpm wxt build -b safari`.

    **If the module is NOT found or throws errors with WXT 0.20.x (fallback — always reliable):**
    Run these two commands:
    ```bash
    pnpm wxt build -b safari
    xcrun safari-web-extension-converter .output/safari-mv3 \
      --project-location ./safari-extension \
      --app-name "LinkedIn HHRH Screener" \
      --bundle-identifier com.linkedin-hhrh.screener \
      --swift \
      --force
    ```

    Use the approach that succeeds. Document which approach was used in the SUMMARY.

    After the build succeeds: confirm the Xcode project directory exists (either `.output/LinkedInHHRRScreener/` or `./safari-extension/`).

    Do NOT attempt to open Xcode or build the native app — just ensure the project files exist.
  </action>
