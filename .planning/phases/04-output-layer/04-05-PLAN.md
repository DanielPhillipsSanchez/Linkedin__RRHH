Note: anthropic.ts uses declarativeNetRequest to inject auth headers. In tests, chrome is
undefined — anthropic.ts already has a pattern of calling chrome.declarativeNetRequest which
will throw unless mocked. Review anthropic.ts carefully: it calls
chrome.declarativeNetRequest.updateSessionRules unconditionally (unlike cortex.ts which guards
with typeof chrome !== 'undefined'). This means tests that call anthropicComplete directly
will need chrome.declarativeNetRequest stubbed.

The simplest approach: stub chrome globally in tests that call anthropicComplete:
```typescript
vi.stubGlobal('chrome', {
  declarativeNetRequest: {
    updateSessionRules: vi.fn().mockResolvedValue(undefined),
    RuleActionType: { MODIFY_HEADERS: 'modifyHeaders' },
    HeaderOperation: { SET: 'set' },
    ResourceType: { XMLHTTPREQUEST: 'xmlhttprequest' },
  },
});
```
Add this stub in the beforeEach of any describe block that calls into anthropicComplete
(refineWithClaude, generateOutreachMessage, handleEvaluate, handleGenerateMessage,
validateStoredCredentials).
